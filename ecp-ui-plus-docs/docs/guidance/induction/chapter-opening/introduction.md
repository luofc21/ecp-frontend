# 入门指南

## 编写目的

本指南旨在帮助新成员快速了解公司业务流程，掌握前端开发相关知识与技能，帮助新成员提高工作效率与产出质量。

## 培训计划

- **第一天**
  - 导师引导认识部门成员，认识项目组其他部门成员；
  - 了解工作职责，了解开发流程及相关工作系统；
  - 学习公司业务架构（技术平台、产品应用、公共服务）；
- **第二天**
  - 学习公司系统架构（微服务、API 网关 、sso 鉴权，nacos）；
  - 学习公司前端技术架构（Web 端、大屏端、移动端）；
  - 学习公司前端相关规范（代码规范、git 提交规范、 UI 规范、组件规范、  组件调用  ）；
- **第三天**
  - 开发环境配置，了解相关工具及插件（编辑工具 vs）；
  - 学习并实操构建前端项目 （通过脚手架从 0-1 搭建 Demo 项目）；
  - 学习使用公司前端现有组件库；
- **第四天**
  - 继续学习并实操构建前端项目（Demo 项目）；
  - 了解前端部署包构建（Jenkins 构建部署包，了解 uni-server、服务注册、服务订阅 ），并实操完成 Demo 项目部署；
- **第五天**
  - 构建的 Demo 项目 Review；
  - 培训考核。

## 课程大纲

课程主要围绕业务与技术两部分开展，课程大纲如下：

- 开篇引导
- 业务篇
  - 技术平台介绍
  - 产品应用介绍
  - 公共服务介绍
- 技术篇
  - 系统架构
  - 技术架构
  - 相关规范
- 实践篇
  - 开发环境配置
  - 手摸手教你构建前端项目
  - 手摸手教你构建前端部署包
  - 复盘点评
- 结语

## 培训考核

- 考核目标
  - 完成培训课程，并讲解所产出的合规合要求Demo项目；
- 考核标准
  - Demo项目逻辑设计与实现，产出代码质量，实际交互效果。

## 培训结果

完成培训后，新成员可快速投入至平台、产品、项目开发中，并高效保质完成工作产出。
