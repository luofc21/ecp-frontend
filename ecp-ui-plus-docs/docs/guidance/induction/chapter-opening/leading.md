# 开篇引导

## 一、工作职责

  从传统软件工程的主要环节定义可以了解到（可行性分析、需求分析、概要设计、详细设计、实现、测试、运行、维护），几乎每一个环节都需要研发人员参与。那么前端开发作为研发人员，工作内容必然不只是coding：你需要了解对应平台、产品业务属性，了解客户对象与使用用户，理解需求、理解业务，融合平面设计、交互设计，通过代码编写，输出符合用户需求的软件产品，并根据用户反馈不断进行维护/迭代。

  为达到以上要求，你必须具备一定的设计能力、实现能力与攻关能力。以下，将围绕这三个能力进行工作职责阐述。

### 设计能力

  作为前端开发人员需要具备的设计能力，不是指能输原型设计、UED设计，而是依据已输出的原型反向推导功能设计是否合理、设计产物是否符合需求文本描述，以及依据已输出的UI判断与原型功能与逻辑设计是否相符、设计是否符合规范等，因此你需要：

> - 参与需求评审，了解即将开发的产品、项目业务及设计、开发范围；
> - 参与原型评审，进一步理解需求设计逻辑，与产品、UED、后端开发及其它团队相关人员对需求的理解达成共识；
> - 参与UI评审，确认设计逻辑、交互逻辑与原型相符，确认UI规范符合规范。

### 实现能力

  作为前端开发人员，最主要的工作是将文本描述、平面设计，通过技术方案设计与代码编写，“翻译”成可靠、可用、可维护的软件产品，而整个产品、项目成果也最直接体现在前端开发的产物上，因此你需要：

> - 在理解业务与需求的基础上，进行前端实现方案设计，必要时需进行设计方案评审，并向相关技术、非技术成员进行方案陈述，并输出相关文档；
> - 根据设计要求，完成前端功能模块代码编写、联调、测试与维护，达成产品、项目的开发目标；
> - 定期进行代码审查和代码优化，提高代码质量与产出质量，实现性能优化。

### 攻关能力

  团队协作开发的过程中，从产品、项目启动到落地的过程中，你将会收到来自内部成员与外部人员反馈的各种问题，因此你需要：

> - 参与组内成员、团队成员的沟通交流，并输出相关记录文档；
> - 及时响应，对反馈的问题进行定位、分析，提出技术解决方案，并完成方案验证。
